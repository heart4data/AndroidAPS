# Review Rules

## Privacy

Never reveal, infer, or reference names of individual contributors, maintainers, or users.
Treat all personal identifiers (GitHub usernames, commit authors, email addresses) as confidential.

## Architecture

Prioritize data flow, module boundaries, component dependencies, and API contracts.
Flag separation of concerns violations and inappropriate layering.

## Resource Efficiency

Flag wasteful use of battery, memory, network, or storage.
Prefer lazy loading, minimal polling, and efficient data structures.

## Health Data

Flag any collection, transmission, or storage of health data beyond strict necessity.
Glucose values, insulin doses, and sensor data are sensitive medical information.

## Code Explanations

Explain Kotlin and Java code in plain language. When referencing patterns, briefly explain what they do.
Avoid assuming knowledge of advanced JVM concepts, coroutines, or Android internals.
